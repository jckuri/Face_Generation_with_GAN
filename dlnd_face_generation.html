Face Generation¶

In this project, you'll define and train a DCGAN on a dataset of faces. Your goal is to get a generator network to generate new images of faces that look as realistic as possible!

The project will be broken down into a series of tasks from loading in data to defining and training adversarial networks. At the end of the notebook, you'll be able to visualize the results of your trained Generator to see how it performs; your generated samples should look like fairly realistic faces with small amounts of noise.

def conv(in_channels, out_channels, kernel_size, stride=2, padding=1, batch_norm=True):
    # Initialize the list of layers.
    layers = []
    # Add a convnet.
    layers.append(nn.Conv2d(in_channels, out_channels, kernel_size, stride, padding, bias=False))
    # Add batch normalization, if required.
    if batch_norm:
        layers.append(nn.BatchNorm2d(out_channels))
    # Return the sequence of layers.
    return nn.Sequential(*layers)

class Discriminator(nn.Module):

    def __init__(self, conv_dim):
        """
        Initialize the Discriminator Module
        :param conv_dim: The depth of the first convolutional layer
        """
        super(Discriminator, self).__init__()

        # complete init function
        # Initialize conv_dim
        self.conv_dim = conv_dim
        # Create convnet 1
        self.conv1 = conv(3, conv_dim, 4, batch_norm = False)
        # Create convnet 2
        self.conv2 = conv(conv_dim, conv_dim * 2, 4)
        # Create convnet 3
        self.conv3 = conv(conv_dim * 2, conv_dim * 4, 4)
        # Create final fully-connected layer
        self.fc = nn.Linear(conv_dim * 4 * 4 * 4, 1)

    def forward(self, x):
        """
        Forward propagation of the neural network
        :param x: The input to the neural network     
        :return: Discriminator logits; the output of the neural network
        """
        # define feedforward behavior
        # Process convnet 1
        out = F.leaky_relu(self.conv1(x), 0.2)
        # Process convnet 2
        out = F.leaky_relu(self.conv2(out), 0.2)
        # Process convnet 3
        out = F.leaky_relu(self.conv3(out), 0.2)
        # Flatten the outputs of convnet 3
        out = out.view(-1, self.conv_dim * 4 * 4 * 4)
        # Process the final fully-connected layer
        return self.fc(out)

Generator¶

The generator should upsample an input and generate a new image of the same size as our training data 32x32x3. This should be mostly transpose convolutional layers with normalization applied to the outputs.

def deconv(in_channels, out_channels, kernel_size, stride=2, padding=1, batch_norm=True):
    # Initialize the list of layers.
    layers = []
    # Add a convnet.
    layers.append(nn.ConvTranspose2d(in_channels, out_channels, kernel_size, stride, padding, bias=False))
    # Add batch normalization if required.    
    if batch_norm:
        layers.append(nn.BatchNorm2d(out_channels))
    # Return the sequence of layers.
    return nn.Sequential(*layers)

class Generator(nn.Module):
    
    def __init__(self, z_size, conv_dim):
        """
        Initialize the Generator Module
        :param z_size: The length of the input latent vector, z
        :param conv_dim: The depth of the inputs to the *last* transpose convolutional layer
        """
        super(Generator, self).__init__()

        # complete init function
        # Initialize conv_dim
        self.conv_dim = conv_dim
        # Create the first fully-connected layer
        self.fc = nn.Linear(z_size, conv_dim * 4 * 4 * 4)
        # Create convnet 1
        self.t_conv1 = deconv(conv_dim * 4, conv_dim * 2, 4)
        # Create convnet 2
        self.t_conv2 = deconv(conv_dim * 2, conv_dim, 4)
        # Create convnet 3
        self.t_conv3 = deconv(conv_dim, 3, 4, batch_norm = False)

    def forward(self, x):
        """
        Forward propagation of the neural network
        :param x: The input to the neural network     
        :return: A 32x32x3 Tensor image as output
        """
        # define feedforward behavior
        # Process the final fully-connected layer
        out = self.fc(x)
        # Adjust the dimensions correspondingly: (batch_size, depth, 4, 4)
        out = out.view(-1, self.conv_dim * 4, 4, 4)
        # Process convnet 1
        out = F.relu(self.t_conv1(out))
        # Process convnet 2
        out = F.relu(self.t_conv2(out))
        # Process convnet 3
        out = self.t_conv3(out)
        # Return the hyperbolic tangent of the final output
        return F.tanh(out)

Discriminator and Generator Losses¶

Now we need to calculate the losses for both types of adversarial networks.

Discriminator Losses¶

  • For the discriminator, the total loss is the sum of the losses for real and fake images, d_loss = d_real_loss + d_fake_loss.
  • Remember that we want the discriminator to output 1 for real images and 0 for fake images, so we need to set up the losses to reflect that.

Generator Loss¶

The generator loss will look similar only with flipped labels. The generator's goal is to get the discriminator to think its generated images are real.

# This is an abstract loss function.
def loss(D_out, real = True):
    # Obtain the batch size.
    batch_size = D_out.size(0)
    # Labels are either ones or zeros depending on whether the loss is real or fake.
    labels = torch.ones(batch_size) if real else torch.zeros(batch_size)
    # Move labels to the GPU, if necessary.
    if train_on_gpu: labels = labels.cuda()
    # Define the criterion for the loss.
    criterion = nn.BCEWithLogitsLoss()
    # Compute the loss.
    return criterion(D_out.squeeze(), labels)

def real_loss(D_out):
    '''Calculates how close discriminator outputs are to being real.
       param, D_out: discriminator logits
       return: real loss'''
    return loss(D_out, real = True)

def fake_loss(D_out):
    '''Calculates how close discriminator outputs are to being fake.
       param, D_out: discriminator logits
       return: fake loss'''
    return loss(D_out, real = False)

def train(D, G, n_epochs, print_every=50):
    '''Trains adversarial networks for some number of epochs
       param, D: the discriminator network
       param, G: the generator network
       param, n_epochs: number of epochs to train for
       param, print_every: when to print and record the models' losses
       return: D and G losses'''
    
    # move models to GPU
    if train_on_gpu:
        D.cuda()
        G.cuda()

    # keep track of loss and generated, "fake" samples
    samples = []
    losses = []

    # Get some fixed data for sampling. These are images that are held
    # constant throughout training, and allow us to inspect the model's performance
    sample_size=16
    fixed_z = np.random.uniform(-1, 1, size=(sample_size, z_size))
    fixed_z = torch.from_numpy(fixed_z).float()
    # move z to GPU if available
    if train_on_gpu:
        fixed_z = fixed_z.cuda()

    # epoch training loop
    for epoch in range(n_epochs):

        # batch training loop
        for batch_i, (real_images, _) in enumerate(celeba_train_loader):

            batch_size = real_images.size(0)
            real_images = scale(real_images)

            # ===============================================
            #         YOUR CODE HERE: TRAIN THE NETWORKS
            # ===============================================
            
            # 1. Train the discriminator on real and fake images
            
            # TRAIN THE DISCRIMINATOR ON REAL IMAGES.
            # Set the gradients to zero
            d_optimizer.zero_grad()
            # Move real_images to the GPU, if necessary.
            if train_on_gpu: real_images = real_images.cuda()
            # Process real images
            D_real = D(real_images)
            # Compute the discriminator loss for real images.
            d_real_loss = real_loss(D_real)
            
            # TRAIN THE DISCRIMINATOR ON FAKE IMAGES.
            # Compute the random vector to generate images.
            z = torch.from_numpy(np.random.uniform(-1, 1, size = (batch_size, z_size))).float()
            # Move z to the GPU, if necessary.
            if train_on_gpu: z = z.cuda()
            # Generate fakes images based on the random vector
            fake_images = G(z)
            # Process fake images
            D_fake = D(fake_images)
            # Compute the discriminator loss for the fake images.
            d_fake_loss = fake_loss(D_fake)
            
            # Compute the total loss for the discriminator.
            d_loss = d_real_loss + d_fake_loss
            # Backpropagate the error
            d_loss.backward()
            # Optimize the discriminator 1 step.
            d_optimizer.step()

            # 2. Train the generator with an adversarial loss
            
            # Set the gradients to zero.
            g_optimizer.zero_grad()
            # Compute the random vector to generate images.
            z = torch.from_numpy(np.random.uniform(-1, 1, size = (batch_size, z_size))).float()
            # Move z to the GPU, if necessary.
            if train_on_gpu: z = z.cuda()
            # Generate fakes images based on the random vector
            fake_images = G(z)
            # Process fake images
            D_fake = D(fake_images)
            # Compute the generator loss for the fake images.
            g_loss = real_loss(D_fake)
            # Backpropagate the error
            g_loss.backward()
            # Optimize the discriminator 1 step.
            g_optimizer.step()
            
            
            # ===============================================
            #              END OF YOUR CODE
            # ===============================================

            # Print some loss stats
            if batch_i % print_every == 0:
                # append discriminator loss and generator loss
                losses.append((d_loss.item(), g_loss.item()))
                # print discriminator and generator loss
                print('Epoch [{:5d}/{:5d}] | d_loss: {:6.4f} | g_loss: {:6.4f}'.format(
                        epoch+1, n_epochs, d_loss.item(), g_loss.item()))


        ## AFTER EACH EPOCH##    
        # this code assumes your generator is named G, feel free to change the name
        # generate and save sample, fake images
        G.eval() # for generating samples
        samples_z = G(fixed_z)
        samples.append(samples_z)
        G.train() # back to training mode

    # Save training generator samples
    with open('train_samples.pkl', 'wb') as f:
        pkl.dump(samples, f)
    
    # finally return losses
    return losses

Question: What do you notice about your generated samples and how might you improve this model?¶

When you answer this question, consider the following factors:

  • The dataset is biased; it is made of "celebrity" faces that are mostly white
  • Model size; larger models have the opportunity to learn more features in a data feature space
  • Optimization strategy; optimizers and number of epochs affect your final result

Answer:

What do you notice about your generated samples?

Generated faces look too asymmetrical to be real. And symmetry is fundamental for beauty. A face lacking symmetry produces too much neuronal resistance in the pattern recognizers of our brains, making them look ugly. The asymmetry of these generated faces is like the body of Frankenstein who is made of body parts from different persons. It's a good analogy. The fundamental problem of these generated faces is that they are made of parts of different faces with different poses and cameras angles, making the task of putting the facial parts together in the right way nontrivial.

In spite of this problem, DCGAN made a great job. Generated faces are totally new and don't resemble any celebrity in this dataset.

I noticed that celebrity faces have different skin colors. In spite of this diversity, DCGAN managed to seamlessly match skin parts without making generated faces look like zebras, which is amazing.

And how might you improve this model?

Perhaps this model needs more neural layers and more cognitive resolution, in order to capture and represent more facial features.

Moreover, the image sizes of the dataset are too small to capture facial features in a good resolution. Perhaps by working with another dataset with higher resolution, the quality of the generated faces could improve.

Another issue is the different perspectives and camera angles of the different faces, in other words, the Frankenstein effect. That problem could be overcome by using 3D point clouds instead of traditional photos. Or by inferring the tridimensionality of faces by using videos instead of photos.

Moreover, by testing different combinations of metaparameters, it is possible to improve the quality of the generated faces a little bit. I tried different combinations of metaparameters. And I obtained the following results:

z_size = 100, 150, 50
The value 50 produced less details.
Whereas, the value 150 worsened the Frankenstein effect.
The value 100 produced better results.

lr = 0.0002, 0.0001
The value 0.0002 was recommended by the DCGAN paper.
But the value 0.0001 produced smoother learning curves.

epochs = 50, 20
The value 20 produced lower quality images. It's like the model lacked more training.
The value 50 produced better results.

batch_size = 64, 128
The value 64 is perhaps too low for batch normalization.
This fact probably made the value 128 a better choice, producing better results.
